# Repo Scan

Repo Scan 是一个用于 Codex 的只读仓库扫描 skill，适合快速研究 GitHub 仓库、浏览器扩展、插件、CLI 工具、开源项目和第三方产品。

它的目标不是直接安装或改代码，而是先基于证据回答：这个项目是做什么的、适合谁、技术栈是什么、安装和依赖最少占多少空间、运行条件是什么、有什么风险、是否值得二次开发，以及下一步应该怎么做。

## 适用场景

- 研究一个 GitHub 仓库的用途、场景和使用方法
- 判断一个工具能不能安装到本机
- 判断项目是否适合二次开发
- 分析项目技术栈、依赖、模型和本地运行条件
- 估算安装使用的最小磁盘占用
- 判断项目是否适配 Trae、Codex、Claude Code
- 评估浏览器扩展或第三方工具能否用 clean-room 方式做类似产品

## 输出内容

Repo Scan 默认输出：

1. 结论
2. 它是做什么的
3. 适合谁 / 适用场景
4. 技术栈
5. 最小空间占用
6. 怎么用 / 运行条件
7. 安装或接入难点
8. 二开价值
9. 风险
10. 建议下一步
11. 证据来源

## 空间占用评估

Repo Scan 会尽量估算：

- 项目本体最小占用：源码、release 包、浏览器扩展包、二进制文件或 Docker image
- 依赖最小占用：语言运行时、包依赖、系统依赖、数据库、浏览器、Docker layer
- 模型/数据最小占用：LLM、OCR、ASR、embedding、向量库、示例数据、缓存或权重文件
- 首次安装最低预留空间：项目本体 + 依赖 + 模型/数据 + 安装缓存的保守估算

如果公开资料不足以给出精确数字，Repo Scan 会标注为 `待确认`，不会编造结果。

## 使用方式

把本目录放入 Codex skills 目录，或创建软链接：

```bash
ln -sf /path/to/repo-scan ~/.codex/skills/repo-scan
```

验证 skill 结构：

```bash
python3 ~/.codex/skills/.system/skill-creator/scripts/quick_validate.py ~/.codex/skills/repo-scan
```

在 Codex 中可以这样使用：

```text
用 repo-scan 研究这个仓库：https://github.com/example/project
```

```text
扫描这个 Chrome 插件，告诉我能不能自己做一个类似工具，先只读评估，不要安装。
```

```text
这个项目能不能在本地运行？技术栈是什么？依赖和模型最少要占多少空间？
```

## 执行边界

Repo Scan 默认只读：

- 不 clone
- 不安装
- 不运行项目
- 不修改本地文件
- 不操作账号

如果需要进入安装、试运行或二次开发阶段，需要用户明确确认。

## 仓库简介

只读扫描 GitHub 仓库、浏览器扩展、插件和开源工具，输出用途、技术栈、空间占用、运行条件、风险、二开价值和下一步建议。
