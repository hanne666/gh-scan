# 更新时间量化评判标准

用于 `gh-stars-organizer` 判断 GitHub stars 的维护状态。

## 时间字段优先级

1. 优先使用 `pushed_at`。
2. 没有 `pushed_at` 时使用 `updated_at`。
3. 两者都没有时标记为 `待确认`。

## 时间分级

| 时间状态 | 判断条件 | 默认处理 |
|---|---|---|
| 活跃 | 0-6 个月内有 push/update | 正常分类，按价值进入 P0/P1 |
| 半年未更新 | 超过 6 个月，未满 1 年 | 降一级；低 star 或用途不明则进 P2 |
| 一年未更新 | 超过 1 年，未满 3 年 | 默认 P2；经典项目、高 star、资料型项目可保留 P1 |
| 三年未更新 | 超过 3 年 | 默认 P3；除非是经典资料、标准库或已完成型项目 |
| 无更新时间 | 无 `pushed_at` / `updated_at` | P2，标 `待确认` |
| archived / disabled | 仓库归档或禁用 | P3 |

## freshness_score

| 条件 | 分数 |
|---|---:|
| 30 天内更新 | 100 |
| 90 天内更新 | 90 |
| 180 天内更新 | 75 |
| 365 天内更新 | 55 |
| 1-3 年内更新 | 30 |
| 超过 3 年 | 10 |
| archived / disabled | 0 |
| 无更新时间 | 20 |

## 降级规则

| 分数 | 规则 |
|---:|---|
| `>= 75` | 不因时间降级 |
| `55-74` | 最多 P1；低 star 或用途不明则 P2 |
| `30-54` | 默认 P2；高 star 或经典资料可 P1 |
| `10-29` | 默认 P3；经典资料可 P2 |
| `0` | P3 |

## 与 star 数结合

| 条件 | 等级倾向 |
|---|---|
| 高 star + 活跃 + 主题相关 | P0 |
| 中高 star + 用途明确 | P1 |
| 低 star + 半年未更新 | P2 |
| 低 star + 一年未更新 | P3 候选 |
| fork + 已收藏主线 | P2 |
| 同名重复项目 | P2 |
| archived / disabled | P3 |
| 超过 3 年未更新 | P3 |

star 数参考：

| star 数 | 含义 |
|---:|---|
| `>= 50000` | 高影响力项目 |
| `5000-49999` | 有明显社区验证 |
| `100-4999` | 普通项目 |
| `< 100` | 新项目或小众项目，默认待复核，除非用户明确需要 |
